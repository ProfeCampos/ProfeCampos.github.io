🧠 El Cerebro de la Operación: El Código

Todo lo que hacemos con Arduino necesita un plan, y ese plan es el código. Este es el "idioma" que usaremos para darle instrucciones a nuestra placa.

Aquí está el código completo para nuestra misión. No te preocupes si no lo entiendes todo al principio, ¡lo analizaremos línea por línea!

Código Arduino - Misión 01: Blink 

/*
 * Misión 01: Blink - Hacer parpadear un LED
 * Por: Profe Campos
 * CECyTEM 05 Guacamayas
*/

// El bloque setup() se ejecuta UNA SOLA VEZ cuando enciendes o reseteas tu Arduino.
// Es perfecto para configurar todo lo que vamos a usar.
void setup() {
  // Aquí le decimos al Arduino: "Oye, el Pin número 13 será una SALIDA de energía".
  // Es como abrir una puerta para que la electricidad pueda salir.
  pinMode(13, OUTPUT);
}

// El bloque loop() es un ciclo infinito. ¡Se repite una y otra vez para siempre!
// Aquí es donde ocurre toda la acción principal de nuestro programa.
void loop() {
  // 1. Encendemos el LED:
  // Enviamos una señal ALTA (HIGH) de 5V al Pin 13. ¡Luz!
  digitalWrite(13, HIGH);

  // 2. Esperamos un segundo:
  // Hacemos una pausa de 1000 milisegundos (que es igual a 1 segundo).
  // El LED se queda encendido durante este tiempo.
  delay(1000);

  // 3. Apagamos el LED:
  // Enviamos una señal BAJA (LOW) de 0V al Pin 13. ¡Oscuridad!
  digitalWrite(13, LOW);

  // 4. Esperamos otro segundo:
  // Hacemos otra pausa de 1 segundo con el LED apagado.
  delay(1000);

  // Y... el loop() vuelve a empezar desde el principio. ¡Para siempre!
}

🔌 Manos a la Obra: El Circuito

Ahora vamos a construir el cuerpo de nuestra operación. Necesitamos darle al código algo físico que controlar.

Componentes Necesarios:

Cant. Componente ¿Para qué sirve?
1 Placa Arduino UNO Nuestro cerebro principal.
1 Protoboard Nuestro campo de construcción.
1 LED (de cualquier color) Nuestro foquito de luz.
1 Resistencia de 220Ω El "guardaespaldas" del LED.
2 Cables jumper Las "venas" que llevan la electricidad.

📝 Instrucciones de Conexión (Paso a Paso):

  1. Coloca el LED: Inserta el LED en la protoboard. ¡OJO! El LED tiene una pata más larga que la otra. La pata larga es la positiva (+) y la corta es la negativa (-).
  2. Conecta el "guardaespaldas": Conecta un extremo de la resistencia de 220Ω en la misma fila de la pata larga (+) del LED. El otro extremo de la resistencia ponlo en una fila vacía.
  3. Lleva la energía: Usa un cable jumper para conectar desde el Pin Digital 13 del Arduino hasta la fila donde dejaste el extremo libre de la resistencia.
  4. Cierra el circuito: Usa otro cable jumper para conectar desde la pata corta (-) del LED hasta uno de los pines GND (Ground o Tierra) del Arduino.

💡 Conceptos Clave de la Misión

  • Resistencia: Imagina que el Arduino le da un "empujón" de 5V a la electricidad, pero el LED solo aguanta un empujón pequeño. La resistencia es como un guardaespaldas que frena ese empujón para que no "queme" al LED. ¡Siempre usa una resistencia con los LEDs!
  • GND (Tierra): La electricidad es como un circuito de carreras: necesita un punto de partida y una meta para poder fluir. GND es la "meta" o el camino de regreso. Sin él, la electricidad no se mueve y nada funciona.
  • HIGH y LOW: Son las señales digitales más básicas. HIGH es la orden para "encendido" (envía 5V) y LOW es la orden para "apagado" (envía 0V).

🚀 ¡Inténtalo Tú Mismo! (Retos)

Ya que eres el jefe, ¿por qué no pruebas a cambiar las órdenes?

  • ❤️ Latido de Corazón: Haz que el LED haga dos parpadeos rápidos seguidos de una pausa larga.
  • 🆘 S.O.S.: El código internacional de S.O.S. es "3 parpadeos cortos, 3 largos, 3 cortos". ¿Puedes modificar el código para que tu LED envíe esta señal de auxilio? (Pista: necesitarás más digitalWrite() y delay()).
  • 🎁 Reto Sorpresa: Modifica los tiempos de los delay() al mínimo posible (por ejemplo, 10 o 20 milisegundos) e intenta observar qué pasa con la luz del LED. ¿Sigue parpadeando o parece estar siempre encendido pero más tenue?
