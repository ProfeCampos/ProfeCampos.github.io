Tomo 3: ¡Tu Ciudad Inteligente! - El Semáforo Inclusivo[cite: 1]

¡Hola de nuevo, inventores urbanos! Hasta ahora hemos encendido luces y aprendido a escuchar botones. Pero en el mundo real, la tecnología tiene un propósito más alto: resolver problemas reales y hacer el mundo más accesible. Hoy nos convertiremos en ingenieros de tráfico urbano.[cite: 1]

Vamos a fusionar lo que ya conocemos para construir un Semáforo Inteligente e Inclusivo: un sistema que detenga el tráfico vehicular cuando un peatón lo solicite, y que además emita alertas sonoras intermitentes para ayudar a cruzar de forma segura a personas con debilidad visual. ¡Prepárate para dar el salto al siguiente nivel de programación![cite: 1]

🧠 El Protagonista de Hoy: El Zumbador Pasivo (Buzzer)[cite: 1]

Para lograr la inclusión vial, nuestro gran aliado será el Buzzer pasivo. A diferencia de un zumbador activo (que solo hace un pitido monótono cuando le das corriente), el buzzer pasivo funciona como un mini altavoz: no emite ningún sonido a menos que tú le envíes una onda cuadrada con una velocidad específica.[cite: 1]

A esa velocidad la llamamos Frecuencia y se mide en Hertz (Hz).[cite: 1] Si le mandas una frecuencia alta (por ejemplo, 1500 Hz), el buzzer vibrará rápido y emitirá un sonido agudo (como un silbato); si le mandas una frecuencia baja (como 300 Hz), vibrará lento y sonará grave (como un motor).[cite: 1] Para controlarlo, usaremos la instrucción estrella: tone(pin, frecuencia, duración);.[cite: 1]

💡 El Gran Dilema: El "Código Espagueti" y el Control del Tráfico[cite: 1]

Imagina la escena: tienes que apagar el verde de autos, encender el amarillo por 2 segundos, luego el rojo; inmediatamente después, apagar el rojo peatonal, encender el verde peatonal y, al mismo tiempo, hacer que el buzzer empiece a pitar de forma intermitente (suena-pausa-suena-pausa) para guiar al peatón.[cite: 1]

Si intentamos escribir todas estas instrucciones secuenciales una tras otra directamente dentro de la función void loop(), nuestro código se convertirá en lo que en el mundo del software llamamos "Código Espagueti": un enredo gigante de líneas, tiempos y retrasos (delay) donde es facilísimo equivocarse y casi imposible encontrar un error. El loop() se volvería tan pesado que el Arduino tardaría en reaccionar cuando alguien presione el botón.[cite: 1]

¿Cómo resolvemos este caos? La respuesta es la Programación Modular mediante Funciones Personalizadas. Así como Arduino ya viene de fábrica entendiendo comandos como digitalWrite() o tone(), tú tienes el superpoder de inventar tus propios comandos. Vamos a empaquetar toda la secuencia del semáforo y los ritmos del buzzer dentro de una función propia llamada activarPasoPeatonal(). De este modo, nuestro void loop() quedará impecable y ultra rápido: solo se dedicará a vigilar el botón. En cuanto detecte que alguien lo presiona, "llamará" a nuestra función para que ejecute la coreografía vial y, al terminar, el control regresará al flujo principal. ¡Es pasar de escribir código a diseñar sistemas![cite: 1]

🔌 El Circuito: Tu Intersección Vial[cite: 1]

Para armar esta maqueta urbana, la organización en tu protoboard será vital.[cite: 1]

Componentes Necesarios:

  • 3 LEDs para autos (Rojo, Amarillo, Verde).[cite: 1]
  • 2 LEDs para peatones (Rojo, Verde).[cite: 1]
  • 5 Resistencias de 220 o 330 ohms (una para cada LED).[cite: 1]
  • 1 Botón pulsador.[cite: 1]
  • 1 Buzzer (Zumbador pasivo).[cite: 1]
  • Cables jumper.[cite: 1]

Instrucciones de Conexión:

  1. Semáforo de Autos: Conecta el LED Verde al pin 10, Amarillo al pin 11 y Rojo al pin 12 (todos con su resistencia hacia la línea de GND).[cite: 1]
  2. Semáforo Peatonal: Conecta el LED Verde al pin 8 y Rojo al pin 9 (con su resistencia a GND).[cite: 1]
  3. El Buzzer Inclusivo: Conecta su pata positiva (+) al Pin Digital 7 y la otra pata directo a GND.[cite: 1]
  4. El Botón Peatonal: Conecta una patita al Pin Digital 2 y la otra patita directo a GND. (Aprovecharemos la resistencia interna de Arduino configurando el pin como INPUT_PULLUP para ahorrar espacio en la protoboard).[cite: 1]

🧠 El Código: Creando Nuestro Propio Comando[cite: 1]

Presta mucha atención a cómo se estructura la función personalizada fuera del setup y del loop, y cómo es invocada cuando el botón cambia de estado.[cite: 1]


/*
 * Misión 03: ¡El Semáforo Inteligente Inclusivo!
 * Descripción: Aprendiendo a programar funciones personalizadas para coordinar
 * un sistema interactivo de tráfico con asistencia auditiva por buzzer.
 * Por: Profe Campos
 * CECyTEM 05 Guacamayas
 */

// Pines del Semáforo de Autos
int cocheVerde = 10;
int cocheAmarillo = 11;
int cocheRojo = 12;

// Pines del Semáforo Peatonal
int peatonVerde = 8;
int peatonRojo = 9;

// Asistentes del sistema
int pinBuzzer = 7;
int botonPeaton = 2;

void setup() {
  // Configuramos pines de iluminación y sonido como SALIDAS
  pinMode(cocheVerde, OUTPUT);
  pinMode(cocheAmarillo, OUTPUT);
  pinMode(cocheRojo, OUTPUT);
  pinMode(peatonVerde, OUTPUT);
  pinMode(peatonRojo, OUTPUT);
  pinMode(pinBuzzer, OUTPUT);

  // Activamos el pull-up interno para el botón (Recuerda: Presionado = LOW)
  pinMode(botonPeaton, INPUT_PULLUP);

  // Estado inicial seguro: Autos avanzan, peatones esperan en rojo
  digitalWrite(cocheVerde, HIGH);
  digitalWrite(peatonRojo, HIGH);
}

void loop() {
  // El loop se mantiene limpio y veloz, vigilando el botón constantemente
  if (digitalRead(botonPeaton) == LOW) {
    // ¡Invocamos nuestro comando personalizado! El programa salta a ejecutarlo
    activarPasoPeatonal(); 
  }
}

// ====================================================================
// 🛠️ NUESTRA FUNCIÓN PERSONALIZADA: ¡Aquí está la magia!
// ====================================================================
// Usamos 'void' porque esta función ejecuta acciones pero no devuelve números.
void activarPasoPeatonal() {
  delay(500); // Pequeña pausa de seguridad antes de iniciar la transición

  // 1. Fase de Advertencia para Autos (Verde -> Amarillo)
  digitalWrite(cocheVerde, LOW);
  digitalWrite(cocheAmarillo, HIGH);
  delay(2000); // 2 segundos para que los autos frenen
  
  // 2. Fase de Detención Total de Autos (Amarillo -> Rojo)
  digitalWrite(cocheAmarillo, LOW);
  digitalWrite(cocheRojo, HIGH);
  delay(1000); // Tiempo de colchón antes de dar paso al peatón

  // 3. Fase de Cruce Peatonal Inclusivo
  digitalWrite(peatonRojo, LOW);
  digitalWrite(peatonVerde, HIGH);

  // El buzzer emite un ritmo constante y seguro (Frecuencia de 800Hz)
  // Ciclo auditivo 1
  tone(pinBuzzer, 800, 300); // Tono medio por 300ms
  delay(600);
  // Ciclo auditivo 2
  tone(pinBuzzer, 800, 300);
  delay(600);
  // Ciclo auditivo 3
  tone(pinBuzzer, 800, 300);
  delay(600);

  // 4. Fase de Advertencia Peatonal (¡El tiempo se agota!)
  // Hacemos que el LED verde parpadee y el buzzer suene más agudo y rápido
  digitalWrite(peatonVerde, LOW);
  tone(pinBuzzer, 1200, 150); // Tono agudo de prisa
  delay(300);
  digitalWrite(peatonVerde, HIGH);
  delay(300);
  
  digitalWrite(peatonVerde, LOW);
  tone(pinBuzzer, 1200, 150);
  delay(300);
  digitalWrite(peatonVerde, HIGH);
  delay(300);

  // 5. Cierre del ciclo y retorno a la normalidad
  digitalWrite(peatonVerde, LOW);
  digitalWrite(peatonRojo, HIGH); // Peatones esperan otra vez
  delay(1000);
  
  digitalWrite(cocheRojo, LOW);
  digitalWrite(cocheVerde, HIGH); // Autos vuelven a circular
  // Aquí termina la función. El Arduino regresa al 'loop' justo donde se quedó.
}

🚀 ¡Inténtalo Tú Mismo! (Retos)[cite: 1]

¡Pon a prueba tu mente de arquitecto de sistemas![cite: 1]

La Alerta de Ceguera del Conductor: Modifica tu función personalizada para que cuando el semáforo de autos pase a Rojo, el buzzer emita un único pitido súper agudo y largo (ej. 2000 Hz por 400 milisegundos) antes de que se encienda el verde peatonal.[cite: 1] Esto avisará acústicamente que los autos se han detenido por completo.[cite: 1]
  2. Ritmo Sincronizado: Intenta modificar la sección de "Fase de Advertencia Peatonal" en el código para que el buzzer y el LED peatonal estén perfectamente sincronizados: cuando el LED se encienda el buzzer suena, y cuando el LED se apague el buzzer haga silencio absoluto.[cite: 1] ¡Observa con atención los delay actuales para lograrlo![cite: 1]
